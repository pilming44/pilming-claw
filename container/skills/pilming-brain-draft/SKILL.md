# Thought Logger

자연어 입력 → 구조화된 마크다운 파일 자동 생성 스킬.

## 저장 경로

```
/workspace/extra/draft/
```

컨테이너 내부 경로이며 호스트의 `/Users/pilming/pilming-brain/draft/`에 매핑된다.
디렉터리가 없으면 생성 후 저장한다.

```bash
mkdir -p /workspace/extra/draft/
```

---

## 실행 절차

### 1. 시각 확정

사용자가 시각을 명시했으면 그것을 사용. 없으면 현재 시각을 사용한다.

```bash
date '+%Y %m %d %H %M %u'
# 출력 예: 2026 04 12 14 35 0  (마지막 숫자: 0=일 1=월 … 6=토)
```

요일 매핑: `0→일 1→월 2→화 3→수 4→목 5→금 6→토`

### 2. 분석 수행

아래 항목을 원문에서 추출·생성한다.

| 필드 | 규칙 |
|------|------|
| `title` | 15자 이내, 명사형 종결. 예: `세관금액 관련 후속 조치 필요` |
| `summary` | 1~2문장. 무엇이 있었고 왜 기록하는지를 담는다 |
| `tags` | 3~5개, 한글 단어, flat 구조 |
| `status` | 할 일이 포함되면 `open` / 순수 감상·메모면 `closed` |
| `맥락` | 이 생각이 나온 배경이나 상황 |
| `핵심 의도` | 사용자가 진짜 원하는 것 |
| `추출된 할 일` | 구체적 액션아이템. 없으면 `없음` |

### 3. 파일명 생성

패턴: `{YYYYMMDD}-{HHmm}-{title-slug}.md`

- `title-slug`: title의 공백을 `-`로 치환, 특수문자(`"` `?` `/` `:` `*` `|` `<` `>` `\`) 제거
- 예: `20260412-1435-세관금액-관련-후속-조치-필요.md`

### 4. 파일 작성

아래 포맷을 **정확히** 준수한다.

```markdown
---
created: {YYYY-MM-DD} {요일} {HH:mm}
title: "{title}"
summary: "{summary}"
tags:
  - {태그1}
  - {태그2}
  - {태그3}
status: {open|closed}
---

## 원문

{사용자가 입력한 자연어 텍스트를 그대로 유지. 절대 수정하지 않는다}

---

## AI 분석

- 맥락: {맥락}
- 핵심 의도: {핵심 의도}
- 추출된 할 일:
  - {할일1}
  - {할일2}
```

### 5. 저장 및 응답

파일을 `/workspace/extra/draft/{파일명}`에 저장한 뒤, 사용자에게 아래 정보를 간결하게 전달한다.

- 저장된 파일명
- title
- status
- 추출된 할 일 목록 (있는 경우)

---

## 주의사항

- YAML frontmatter의 `title`과 `summary` 값은 반드시 쌍따옴표(`"`)로 감싼다.
- `## 원문` 섹션은 어떤 경우에도 원본 그대로 보존한다. 오탈자도 수정하지 않는다.
- 추출된 할 일이 여러 개면 줄을 나눠 각각 적는다.
- tags는 계층 없이 flat하게, 한글 단어로만 작성한다.
- 파일 저장 후 내용을 채팅창에 전체 출력하지 않는다. 요약 정보만 전달한다.

---

## 예시

**입력:**
> 오늘 세관에서 연락 왔는데 신고 금액이 맞지 않는다고 함. 다음 주까지 수정 신고서 제출해야 하고, 담당자 이름은 김철수 주임. 연락처 확인해서 다시 전화해야 할 것 같음.

**생성 파일명:** `20260412-1435-세관금액-관련-후속-조치-필요.md`

**생성 내용:**
```markdown
---
created: 2026-04-12 토 14:35
title: "세관금액 관련 후속 조치 필요"
summary: "세관으로부터 신고 금액 불일치 통보를 받아 다음 주까지 수정 신고서를 제출해야 하는 상황을 기록한다."
tags:
  - 세관
  - 서류제출
  - 연락
status: open
---

## 원문

오늘 세관에서 연락 왔는데 신고 금액이 맞지 않는다고 함. 다음 주까지 수정 신고서 제출해야 하고, 담당자 이름은 김철수 주임. 연락처 확인해서 다시 전화해야 할 것 같음.

---

## AI 분석

- 맥락: 세관 신고 금액 불일치로 인해 행정 처리가 필요한 상황
- 핵심 의도: 기한 내 서류 제출과 담당자 연락을 잊지 않기 위한 기록
- 추출된 할 일:
  - 김철수 주임 연락처 확인
  - 수정 신고서 작성 및 다음 주까지 제출
  - 담당자에게 재연락
```
